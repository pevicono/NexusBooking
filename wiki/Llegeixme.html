NexusBooking

Llegeix-me: Guia de posada en marxa des de 0

1. Objectiu

Aquesta guia explica com arrancar tot el projecte NexusBooking des de zero: base de dades, backend, client desktop i client mobile, amb comprovacions bàsiques perquè validis que tot funciona abans de fer demo o desenvolupament.

Sistema operatiu recomanat: Windows (els exemples usen PowerShell i scripts .cmd).

2. Prerequisits

Eina Versió recomanada Ús
Docker Desktop Actual PostgreSQL + pgAdmin
Java 21+ Backend i desktop
Maven 3.8+ Build i execució backend/desktop
Android Studio + SDK API 26+ App Android
Si PowerShell et bloqueja scripts, usa sempre db.cmd (ja aplica bypass de política internament).

3. Estructura ràpida del projecte

NexusBooking/
├── database/   # Docker, scripts i dades demo
├── backend/    # Spring Boot API REST + JWT
├── desktop/    # JavaFX client (USER + ADMIN)
├── mobile/     # Android (Kotlin + Compose)
└── config.properties  # URL base del desktop

4. Arrancada mínima (DB + Backend)

  1. Obre terminal a l’arrel del repo NexusBooking/.
  2. Arranca contenidors de base de dades:
cd database
.\db.cmd start
  1. (Opcional) Carrega dades demo:
.\db.cmd seed
  1. Arranca el backend:
cd ..\backend
mvn spring-boot:run

Comprovació

ServeiURL
APIhttp://localhost:8080
Swaggerhttp://localhost:8080/swagger-ui.html
pgAdminhttp://localhost:5050

5. Executar el client desktop

Configuració URL backend

Edita el fitxer config.properties a l’arrel del projecte:

api.base.url=http://localhost:8080

Execució

cd desktop
mvn javafx:run

Què validar en obrir

  • Login i registre funcionen.
  • Amb usuari USER: dashboard, reserves, grups i incidències.
  • Amb usuari ADMIN: backoffice (usuaris, instal·lacions, reserves, grups, incidències, calendari).

6. Executar el client mobile (Android)

URL backend

  • Emulador Android: usa http://10.0.2.2:8080 (per defecte).
  • Dispositiu físic: canvia la base URL a mobile/app/src/main/java/com/example/nexusbooking/mobile/data/remote/ApiService.kt amb la IP del teu PC.

Execució

cd mobile
./gradlew installDebug

Alternativa: Android Studio -> Run app.

8. Comandes útils (operació diària)

Database

  • .\db.cmd status
  • .\db.cmd logs
  • .\db.cmd clear
  • .\db.cmd reset
  • .\db.cmd stop

Backend/Tests

  • cd backend
  • mvn test
  • mvn spring-boot:run

9. Resolució de problemes

No connecta amb backend

  • Comprova que backend està aixecat a localhost:8080.
  • Desktop: revisa config.properties.
  • Mobile: revisa base URL (10.0.2.2 en emulador).

Base de dades no arrenca

  • Docker Desktop ha d’estar iniciat.
  • Port 5432/5050 lliures.
  • Executa cd database i .\db.cmd status.

Dades incoherents per proves

  • Per tornar a estat net: .\db.cmd clear + .\db.cmd seed.
  • Per reconstrucció completa: .\db.cmd reset.
